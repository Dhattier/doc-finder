Introduction:
Cloud computing offers flexibility, dynamic scalability, and cost efficiency: however, these benefits are not without tradeoffs in our exposure to cyber threats. Transitioning to OpEx and cloud infrastructure requires us to evaluate and understand the complexity of the shared responsibility model and mitigate the risks tied to identity management, unauthorized access, and regulatory compliance. As a finical provider it is crucial that we understand the threat landscape so that we do not lose our customersĺ trust.

Credential Theft and Phishing Attacks:
Credential theft, often through phishing campaigns, is one of the most common forms of threats when using cloud services. According to Googleĺs Threat Horizons Executive Snapshot (2021), attackers linked to a Russian state-sponsored group named APT28 launched a phishing attack targeting over 12,000 Gmail users. This campaign aimed to trick users into logging into fake credential portals, which allow the attackers to harvest the credentials of victims and use these stolen credentials to access the victimsĺ cloud systems.
These stolen credentials can be used to access cloud resources, steal stored data, and disrupt services. The Cloud Security Alliance notes that weak identity and access management configurations and insecure interfaces are among the most exploited vulnerabilities in cloud environments, often serving as the entry point for phishing campaigns and privilege escalation (Cloud Security Alliance, 2024).

Mitigation Strategy for Credential Theft and Phishing Attacks:
Mitigating credential theft begins with adopting identity and access management (IAM) practices. Amazon Web Services recommends enabling multi-factor authentication for all accounts, using roles instead of static credentials, and enforcing least-privilege access (AWS IAM Best Practices, n.d.). Microsoft Azure recommends implementing role-based access control and conditional access policies (interestingly they mention using conditional access to block legacy authentication protocols) (Microsoft, 2025). Google suggests protecting against compromised identities with context-aware access by evaluating multiple risk signals before granting access (Google Cloud, 2021). The National Security Agency recommends federated identity solutions, session timeouts, and device posture assessments to control access and ensure credential safety (NSA, 2024).

Mitigation Strategy for Resource Exploitation for Crypto Mining
Vulnerability management tools like Googleĺs Container Analysis and Web Security Scanner can help identify exploitable weaknesses (Google Cloud, 2021). Monitoring resource utilization and setting up alerts for abnormal consumption through dashboards can provide early warning signs of abuse. Auditing coding repositories for exposed credentials and setting up protocols to prevent leaked credentials is important as well as these can and have been accidentally published to platforms such as GitHub. Automated logging and system isolation to detect anomalous activity before it can escalate is also important (ENISA, 2021). 

GDPR Compliance and Regulatory Fines:
GDPR grants individuals in the European Union the right to data portability which pressures organizations to offer transparent, accessible, and secure mechanisms for personal data transfer (Portuondo, 2021). GDPR violations carry significant financial impact for non-compliance. Amazon was fined 746 million euro in 2021 for violation of GDPR (CNET, 2021). 
Mitigation Strategy for GDPR Compliance and Regulatory Fines
We need to work with cloud providers that offer compliance aligned architectures which enforce regional data residency and provide end-to-end encryption. Transparent data portability mechanisms, such as user-controlled dashboards and exportable formats, are essential to meeting Article 20 requirements (Portuondo, 2021). While GDPR only applies to users in the EU due to the complexity of managing multiple standards for users in different jurisdictions it will be best to adhere to the most restrictive regulation across users. This will also ensure that no European userĺs data is accidentally treated in a manner that is not GDPR-compliant.

Conclusion:
Cloud computing introduces a threat landscape where credential theft, resource exploitation, and regulatory risks and fines under legalization like GDPR present serous concerns to mitigate. By using a proactive strategy that incorporates IAM best practices, real-time monitoring, and legal compliance planning, we can safely leverage cloud computing and significantly reduce our exposure to risk.
